Валидaция кода html и css

Существует такое понятие, как валидация или валидность кода. Валидация - это процесс тестирования кода на отсутствие ошибок и его соответствие принятым стандартам, которые задаёт World Wide Web Consortium, W3C. Код, который не содержит ошибок, называется валидным.

Проверить валидность html и css кода сайта не составит труда, для этого есть специальные сервисы:

  • validator.w3.org для html;
  • jigsaw.w3.org/css-validator для css.

Зачем нужна проверка валидности html и css кода:

  • Валидность кода сайта позволяет правильно отображать сайт в разных браузерах т.е. вся визуальная и функциональная составляющая сайта отображается и работает правильно. Отображение сайта на экране монитора происходит следующим образом: браузер читает исходный код веб-страницы, расшифровывает его, и результат отображается на вашем экране. Некоторые браузеры, наткнувшись на ошибки код сайта, самостоятельно их исправляют, поэтому страницы с невалидным кодом в таких браузерах будут нормально отображаться. Другие же не исправляют ошибки, и отображают страницу как есть. И в таких браузерах сайт с невалидным кодом будет выглядеть "криво".

  • Поисковые системы эффективнее работают с валидным кодом, а роботы поисковых систем отдают большее предпочтение такому сайту, а значит этот сайт будет находиться выше в поисковой выдаче.

  • Сайт, код которого не содержит ошибок, обрабатывается браузерами быстрее.

  • Валидатор поможет вам проверить вложенность и правильность закрытия тегов, а также выявит ошибки в синтаксисе css, которые вы сами могли не заметить.

  • Ещё одна причина - субъективный подход. Валидный код - это показатель профессионализма человека создающего сайт. Есть люди, стремящиеся писать код аккуратно качественно, и есть те, кто не следит за качеством. Кому отдаст работу хоть немного разбирающийся в деле заказчик - ответ очевиден.

Следует отметить, что валидный код сам по себе еще не гарантирует 100% качество, т.к. большое значение имеет структура кода. При неправильной структуре даже с полностью валидным кодом страница может отображаться неправильно. А иногда возникают ситуации, когда по тем или иным причинам (для решения каких-либо специфических задач или проблем) разработчиком намеренно пишется невалидный код, поэтому в его оценке не все так просто, как может показаться на первый взгляд.
